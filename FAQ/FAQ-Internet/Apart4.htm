4.1.0.- WWW

4.1.1.- Qué es WWW ?

    World Wide Web es como se conoce el resultado del "protocolo de transmisión de HiperTexto", que gracias a sus "enlaces" intra-extra documento, forman la "tela de araña" que muchos conocen (equivocadamente) como "Internet". Ahi se encuentra/busca, tanto información personal, como publicidad, formularios, etc. Actualmente es una de las mayores fuentes de "información" de la red, despues del FTP.

4.1.2.- Qué es www-mail ?

    Pues consiste en que por ejemplo un determinado fichero que está en una página web que te va muy lenta, poder pillarlo de tu buzón de correo donde te lo habrá dejado este servidor de www-mail, con lo que esto supone de aumento de velocidad y que puede ser lo máximo que de tu modem. Para información más detallada consulta La guia del D.Bob (7.1.1)

4.2.0.- FTP

4.2.1.- Qué es anonymous FTP ?

    El FTP (File Transfer Protocol) es un servicio que funciona con el protocolo TCP/IP y que permite acceder a un servidor de este sistema de intercambio para recibir o transmitir archivos de todo tipo. En Internet existen millones de ficheros distribuidos en miles de ordenadores, que pueden ser copiados libremente usando FTP.

    Estos ficheros pueden ser documentos, textos, imagenes, sonidos, programas, etc., conteniendo todo tipo de datos e información. El procedimiento mediante el cual se accede a un ordenador para copiar ficheros en forma libre y sin restricciones, se conoce con el nombre de FTP anónimo. Como es obvio, en este caso no se necesita una contraseña (password) para entrar en el ordenador remoto, basta con poner como identificador la palabra anonymous y como password, la direccion de correo electronico.

    Consejos para usar FTP: En los subdirectorios suelen haber ficheros cuyo nombres son similares a readme o index. Es conveniente transferirlos y leerlos, puesto que suelen dar una idea del contenido de los directorios o indicaciones utiles acerca de lo que se puede obtener en el anfitrión. Los sitios que ofrecen el servicio de FTP anónimo, pueden atender a un numero limitado de usuarios al mismo tiempo; por lo que muchas veces no puede realizarse la conexión apareciendo el correspondiente mensaje del sistema remoto.

4.2.2.- Comandos del FTP

*Nota: Estos comandos corresponden la FTP.exe incluido en los sitemas Win y no pretende ser una lista exahustiva pero si común a todos los sistemas (Dos, Win, OS/2, Linux, etc..)
 
  • !    Shell al DOS. Permite salir al DOS temporalmente. Escribiendo EXIT en el prompt del DOS volvemos al Ftp.
  • ?    Ayuda. Muestra la lista de comandos del programa. Si se pone este comando seguido de un comando aparece la ayuda de ese comando.
  • append    Este comando permite continuar con un download que ha sido cortado brúscamente.
  • ascii    Establece que la transferencia se realice en modo Ascii. Se emplea para transferir textos.
  • bell    Activa o desactiva un beep cuando finalicen las operaciones de trasnferencia (por ejemplo cuando terminemos de bajar un fichero). Por defecto está inactiva.
  • binary    Establece que la transferencia se realice en modo binario. Las transferencias de ficheros deben realizarse en este modo.
  • bye    Terminar la sesión de Ftp y salir del programa.
  • cd    Entrar a un directorio remoto.
  • close    Cerrar la sesión de Ftp.
  • delete    Borrar un fichero remotamente.
  • debug    Activar/desactivar el modo debug :?
  • dir    Listar el directorio activo en el Ftp al que estamos conectados.
  • disconnect    Lo mismo que close.
  • get    Recibir un fichero.
  • glob    Activar/desactivar la expansión de los caracteres en los nombres de los ficheros locales :?
  • hash    Activar/desactivar '#' para cada buffer trasnferido :?
  • help    Lo mismo que ?.
  • lcd    Cambiar de directorio en el disco duro local. Con este comando podemos indicarle al programa dónde dejar los ficheros que nos traigamos.
  • literal    Este comando es MUY INTERESANTE. Permite ejecutar comandos de forma remota en el ordenador al que estemos conectados por el puerto de Ftp. Normalmente con 'literal ?' o 'literal help' obtendremos un listado de los comandos que se nos permite ejecutar en la máquina remota.
  • ls    Listar el directorio activo completo en la máquina remota.
  • mdelete    Como delete pero para múltiples ficheros.
  • mdir    Como dir pero para múltiples ficheros.
  • mkdir    Crear un directorio de forma remota.
  • mls    Como ls pero para múltiples ficheros.
  • mput    Como put pero para múltiples ficheros.
  • open    Con este comando abriremos una conexión a una máquina remota por Ftp.
  • prompt    Forzar respuestas interactivas en múltiples comandos :?
  • put    Enviar un fichero.
  • pwd    Nos dice cuál es el directorio activo en la máquina remota.
  • quit    Como bye.
  • quote    Lo mismo que literal.
  • recv    Lo mismo que get.
  • remotehelp    Es lo mismo que escribir 'literal help'.
  • rename    Cambiar el nombre de un fichero.
  • rmdir    Borrar directorio remotamente.
  • send    Enviar un fichero.
  • status    Muestra dos o tres líneas con el estado de los flags que podemos controlar del programa: tipo de trasnferencia (ascii o binaria), si está activado o no el pitidillo de final de operación (bell), ...
  • trace    Entrar en modo de traceado de paquetes. Para salir volver a escribir trace :?
  • type    Para establecer el tipo de trasnferencia. Type ascii es equivalente a la orden ascii y type binary al comando binary.
  • user    Cambiar el nombre de usuario con el que hemos accedido al Ftp.
  • verbose    Activar/desactivar este flag :?

    •  

     

    Una sesión típica podría ser:
     

  • ftp> lcd c:\auxiliar              -> Para que lo que traigamos se quede en ese directorio de nuestro disco duro.
  • ftp> bell                            -> Para que si estamos haciendo otra cosa mientras se realiza la trasnferencia el programa nos avise cuando termine.
  • ftp> open bdd.unizar.es      -> Para comenzar la conexión.

    •  

     

            Normalmente ahora se nos pide el nombre de usuario y el password. Lo normal será contestar anonymous a ambas preguntas.
     

  • ftp> dir                    -> Para ver lo que hay en el directorio activo.
  • ftp> cd libre             -> Para entrar en el directorio /libre/.
  • ftp> dir                    -> Para ver lo que hay en el directrio activo.
  • ftp> binary               -> Para poder traernos ficheros.
  • ftp> get foto.gif        -> Coger el fichero foto.gif.
  • ftp> close                -> Terminar la conexión.
  • ftp> bye                  -> Salir del programa

    4.3.3.- Qué es una LISTA de correo ?

        Una lista de correo o lista de distribución es un sistema que permite la distribución de mensajes a un conjunto de personas interesadas en un tema concreto (el de la lista). Cada subscriptor de la lista tiene derecho a enviar mensajes (por correo-e) a una direccion especifica de la lista. Desde alli los mensajes se redirigen a todos los miembros de la lista quienes, a su vez, pueden expresar sus opiniones de la misma forma, originandose asi un dialogo entre todos los componentes de la lista. Algunas listas son moderadas, lo que significa que los mensajes son filtrados por el encargado de la misma antes de ser redirigidos a todos los miembros del sistema.

        Links para encontrar una determinada lista:

    4.3.4.- Cómo funciona el PGP ?

        PGP (Pretty Good Privacy) es un programa (o algoritmo) para encriptar los mensajes que mandas a través de InterNet. El programa genera un par de llaves (o claves), una es privada y la otra es pública. Esta última es la que se manda a las personas con las que te carteas via correo-e. Cuando alguien te quiera mandar un mensaje encriptado lo codifica con tu llave pública y luego tu lo descodificas con tu llave privada.

    4.3.6.- Me han avisado de un virus que se activa al abrir el e-mail

        Esto es *MENTIRA*, no hay virus que se pueda transmitir por abrir un e-mail; un virus es un programa, y como tal ha de estar enganchado a algo, por ejemplo, a un ejecutable o a un documento Macro Word. Como programa que es, no se puede ejecutar por mirarse ¿A quien se le ejecuta el Command&Conquer por hacer un dir en su directorio?.
        NO se puede transmitir en un mail e infectarse al leer el mail, porque *ES TEXTO*.  Lo UNICO que podría suceder es que hubiese un documento de Word o un fichero attacheado que se encontrase infectado de un virus, pero para infectarse uno, debería de O abrir ese documento de Word O ejecutar ese fichero. NO HAY otra manera de infectarse, y JAMAS mediante la simple apertura de un mail.

    4.5.4.- Qué diferencia hay entre Kick y Kick/Ban ?

        Pues que si le haces un kick pues simplemente le hechas y cuando vuelva a poner /join #canal_que_te_de_la_gana pues vuelve a entrar, pero si le baneas tambien (ban) pues no puede volver a entrar en el canal a menos que le vuelvas a desbanear.

    4.5.5.- Qué es un netsplit ?

        Split, del ingles literalmente, corte, partido, en el caso del irc, cuando se dice que ha habido un netsplit, quiere decir que uno o varios servidores de una determinada red (undernet por ejemplo) se han desincronizado separando a los tertulianos en funcion de los servidores a los que estaban conectados. desde tu punto de vista los que estan en otro servidor parece que se caen y desde su punto de vista el que se cae eres tu.

    4.5.6.- Qué es un BOT ?

        Un BOT es un _programa_ no un _script_ para un _programa_. Este programa se contecta al irc (automaticamente) igual que tu te conectas (manualmente) con un programa cliente de irc. Estos programas suelen apropiarse de los canales (haciendose OPs de estos mediante netsplits) y puedes ser controlados remotamente mediante comandos. Es muy parecido a los scripts para clientes irc pero mas complejos y mucho mas potentes....

    4.5.7.- Qué es backdoor ?

        Una puerta trasera. En los scripts son codigos o comandos introducidos en el script de forma que cuando alguien te mande el codigo que activa esa backdoor tu script hara algo en contra de tu voluntad, como dar Op a alguien, borrarte algun fichero del disco duro, dar acceso a alguien a tu disco duro, enviar una lista de archivos de tu disco duro a alguien...

    4.9.0.- WAIS

    4.9.1.- Qué es WAIS ?

    Wide Area Information Server, (Busquedas en documentos)

        Las herramientas clásicas utilizadas para buscar información en Internet, presentan algunas limitaciones en lo referente a buscar texto dentro de documentos en equipos remotos. WAIS acrónimo de Wide Area Information Servers, es un sistema diseñado para que el usuario pueda realizar consultas en una amplia red de bases de datos existentes en distintos servidores públicos.(Bibliotecas, Universidades..)

        Es una especie de gestor de base de datos para encontrar documentos basándose en palabras clave, está desarrollado para obtener información a partir de bases de datos distribuidas por todo Inet, cuando se utiliza Wais, éste se conecta a un servidor que hace un "barrido" de indices en busca de la información requerida y después construye un indice con los resultados.

        Se puede usar un programa cliente WAIS de uso público, por ejemplo el de Thinking Machines. Para ello se puede establecer una conexión Telnet, ejecutando telnet quake.think.com, como login "wais", Term=vt100.

        Una cuestión fundamental en WAIS es que el usuario tiene que tener una cierta idea del sitio en el que está lo que busca. De otro modo, dado el gran número de servidores y bases de datos existentes, la búsqueda puede demorarse bastante tiempo. Esta se realiza mediante introducción de palabras, sin importar el orden, que se desean utilizar como referencia para la búsqueda. Se puede conseguir el programa winWAIS con FTP anónimo a ftp.ksc.nasa.gov/pub/winvn
     

    4.10.0.- TELNET

    4.10.1.- Qué es TELNET ?

        TELNET es el protocolo de "conexión" a otro ordenador, de hecho la mayoria de los servicios posteriores, se basan en telnet (pe. FTP, HTTP). Haciendo telnet a una máquina, ejecutas programas en ella, recibiendo tu la entrada/salida de los datos.

        Bob Rankin dice textualmente: "Mucho antes de que la Telaraña y todo el resplandor de sus multimedios fueron una indicación visual siquiera en el radar del Internet, los ciudadanos sabios del Internet estaban utilizando una herramienta basada en texto llamada Telnet para hacer conexión con las maravillas del mundo en-línea. Pero hoy, muchos surfeadores del Internet, quienes no han escuchado hablar del telnet, están perdiendo algo bueno"

        Las direcciones TELNET suelen tener el formato del nombre de dominio "maquina.remota.es" o de dirección IP "194.106.2.150" y pueden ir acompañadas de un número al final (el número del puerto) si no se nos proporciona el puerto se asume que el utilizado es el correspondiente al protocolo telnet por defecto, el 23. Una direccion típica sería: "maquina.remota.es 2010"

    4.10.2.- Qué puedo hacer con TELNET ?

        Por telnet se pueden utilizar TODO tipo de servicios, haciendo telnet a la máquina y puerto correspondientes segun cada caso. Por ejemplo si queremos utilizar
    el servicio POP de nuestro ISP para ver el correo que tenemos, haremos telnet a la maquina POP por el puerto de este protocolo, el 110. También podemos consultar grandes bases de datos e incluso acceder a servicios GHOPER o WWW, muy util si no tenemos acceso a estos servicios por la via normal.

    4.10.3.- Cómo se hace TELNET ?

    Ejecutando un programa cliente de telnet, prácticamente cualquier sistema operativo lleva uno incluido de serie. Por lo tanto si nos proporcionan la dirección telnet "maquina.remota.es 2010" hariamos lo siguiente: (puede variar segun sistemas):
    • Tecleamos en la linea de comandos "TELNET maquina.remota.es 2010"  (En otros sistemas teclearemos "TELNET" y después "OPEN maquina.remota.es 2010" ) con lo que veremos algo parecido a esto:
      • telnet MAQUINA.REMOTA.ES 2010
      • Trying 130.132.21.53 Port 2010 ...
      • Connected to MAQUINA.REMOTA.ES
      • Escape character is ...

        •  
    • Esto nos dice más o menos que está intentando conectar con la dirección, nos devuelve la dirección IP, se conecta, y nos dice cual es el "caracter escape".
    • Una vez hemos conectado se nos pide un "login" y/o "password" para entrar a la máquina remota. En algunos casos podremos conectar a la maquina remota con el login "guest" (invitado) pero la mayoria de las veces deberemos saber el login antes de conectarnos.
    • El siguiente paso es configurar la emulación de terminal, es decir, decirle al sitio remoto como queremos que nos muestre los datos en nuestra pantalla. La configuración más común es la VT100, que es la estandar para las comunicaciones basadas en terminales. (algunos clientes telnet configuran ellos sólos la emulación).
    • El último paso (después de haber utilizado el servicio es salir ;-) Como las pulsaciones de tecla no las hacemos "realmente" en nuestra máquina, sino en la máquina remota, necesitamos el "caracter escape" que se nos dió al conectar para pasar al "modo comando" ( habitualmente teclas control + paréntesis derecho ).

      •  

       

      Comandos disponibles:

      CLOSE         Termina la conexión TELNET con la maquina remota y vuelve al "modo comando" (si empezamos alli) o sale de TELNET.
      QUIT            Sale del programa TELNET; si estás conectado a una máquina remota, QUIT te desconecta y luego sale de TELNET.
      SET ECHO   Si no ves lo que estás escribiendo, o si escribes y ves doble, este comando soluciona el problema.
      OPEN           Abre una conexión a una máquina remota.

      Nota: Al presionar las teclas Enter o Return, sales del modo comando TELNET y vuelves a la sesión TELNET.

    4.10.6.- Qué es HYTELNET ?

    La herramienta HYTELNET, desarrollada por Peter Scott, de la Universidad de Saskatchewan (Canadá), es una base de datos que contiene una lista con más de 1.000 direcciones de sistemas de acceso público, como catálogos de bibliotecas, bases de datos y servidores Gopher y Archie. Ofrece información de los servidores, su ubicación (URL) y datos como la palabra que hay que teclear para acceder (login) y la forma de salir del servidor.

    4.11.0.- Buscadores

    Buscar información es una de las cosas más importantes que se pueden hacer en Internet. De hecho cuando estamos conectados, *siempre* estamos buscando algo: un documento, un web, un programa, un gráfico, el e-mail de algún conocido, etc.. Si no sabemos como localizar lo que queremos de la red estaremos perdidos. Saber *como* buscar algo en la red evita hacer preguntas estúpidas en los foros de usenet (news), en los canales de IRC, etc.. sencillamente saber buscar información por uno mismo es lo que nos diferenciará de esa subespecie que pulula por la red, que tanto abunda y de la que evitaré pronunciar su adjetivo en este FAQ }:)

    Buscadores los hay de muchos tipos, de hecho algunos de los servicios tratados en los apartados anteriores no son mas que buscadores de información, el Archie es un servicio buscador de ficheros, el Finger busca información de usuarios, el Wais hace busquedas en documentos, etc.. Los buscadores de los que hablaré aqui son los potentes buscadores del web, que se diferencian "principalmente" en dos tipos: los "Motores de busqueda" y los "Indices de información". Digo "principalmente" pues también existen en el Web buscadores de news, de e-mails, e incluso buscadores especializados por temas.

    4.11.1.- Motores de Búsqueda

    Son los más poderosos, ya que indexan *todo* lo que sus potentes e incansables robots encuentran en el web y por tanto la información que contienen es inmensa y además actualizada. La única desventaja de este tipo de buscadores es precisamente esa, la cantidad de información que devuelven en una busqueda simple puede ser abrumadora.

    4.11.1.1.- AltaVista

    http://www.altavista.magallanes.net/

    De entre los motores de búsqueda AltaVista es simplemente *el mejor*. Por cantidad de información y por sus potentes herramientas de búsqueda. Sin contar con que hay versiones para multitud de idiomas e incluso ofrece traducción automática entre ellos.

    Bob Rankin dice textualmente: "Alta Vista ostenta la base de datos más grande en la Web, con más de 100 millones de páginas, y puesto que ellos ponen en un índice 10 millones de páginas por día, es probable que sea la más nueva también. Pero dado que catalogan cada palabra de cada página en la Web, el espantoso resultado de "25,679 documentos corresponden a su interrogante" probablemente aparecerá inesperadamente si Uds. no especifican  su búsqueda cuidadosamente."

    4.11.1.2.- Cómo utilizo la Búsqueda Avanzada de AltaVista ?

    Como Bob Rankin ya escribió esto por mi hace algún tiempo, me limito a copiar sus palabras O:) Y os recomiendo las leais con atención, os serán de una ayuda inestimable ;-)

    Aquí está un pequeño secretito para Uds.:  No se molesten en pulsar el botón de "Búsqueda Avanzada" ("Advanced Search") en Alta Vista. Pueden hacer unas cosas realmente asombrosas sin dejar la página principal de AV. ¡En efecto, hasta la gente que desarrolla AV usa la "búsqueda sencilla de AV" ("simple search") casi exclusivamente! Aquí están unos secreticos para buena sintonización de sus búsqueda en AV:

    1) Usen los signos de operación "+" y "-"

    Coloquen al principio de sus palabras de búsqueda un "+" para indicar que ellas tienen que aparecer en una página para que sean consideradas un acierto, y usen "-" para excluir páginas.

    Por ejemplo:

        +chocolate +tortugas

    Encontrarán únicamente páginas que contengan AMBAS palabras.  Si Uds. omiten el "+" AV devolverá unas páginas acerca de chocolate, otras acerca de tortugas, además de páginas que contienen ambas palabras. En este ejemplo, la diferencia es 25414 contra 2467 que corresponden. Con agregar un término de exclusión como se muestra abajo, el número de aciertos baja a 1869.

        +chocolate +tortugas -maní

    2) Usen comillas para frases

    Bueno, 1800 que corresponden es todavía un poco demasiado para asimilar, pues bien, vamos a subir el calor en AV.  Si Uds. quieren realmente encontrar Tortugas de Chocolate (chocolate Turtles) (y no sólo páginas con esas dos palabras) pónganlas entre comillas. ¡La búsqueda

    +"chocolate tortugas" -maní

    produce sólo 49 aciertos!  Ahora estamos en un nivel bastante bajo donde Uds. pueden mirar cada documento que corresponde sin gastar horas.

    3) Usen las palabras claves de "host" ("servidor") y  "domain" ("dominio")

    ¿Alguna vez han querido buscar una palabra específica en un sólo sitio? Intenten con una búsqueda como ésta:

        +"chocolate" +host:ama.org

    Esto le cuenta a AV que encuentre artículos acerca de chocolate publicados sólo por la Asociación Médica Americana, y produce sólo seis aciertos. Uds. pueden excluir un sitio o un dominio entero de una búsqueda:

        +"Election reform" -host:whitehouse.gov ("reforma de las elecciones")
        +"human rights" ("derechos humanos") -domain:cn

    El primer ejemplo excluye documentos publicados por un sitio, (la Casa Blanca de los EE.UU.) y el segundo elimina todos los documentos de un dominio entero (el país de china).

    4) Usen la palabra clave "title" ("título")

    Si Uds. están buscando páginas con un tema específico, en lugar de páginas que sólo contengan ciertas palabras, traten de limitar su búsqueda así:

        title:"chocolate adiction" ("adicción al chocolate")

    Esto con frecuencia suprime aciertos no deseados.  El hecho que alguien se moleste en categorizar su página con una clave de <TITLE> les  podría ayudar a conseguir mejor calidad de documentos que corresponden.

    5) Usen la palabra clave "image" ("imagen")

    ¿Están buscando una foto o un icono en especial?  Intenten algo como esto:

        image:truffle.gif
        +image:comet* +host:nasa.gov

    El primero podría ser obvio, el segundo usa un meta-carácter para encontrar una imagen cuyo nombre empieza con "comet", sea GIF o JPEG, pero solamente en el sitio de la NASA.

    6) Usen la palabra clave "link" ("eslabón") y "url" ("Localizador Uniforme de Recursos o LUR")

    La palabra clave "link" encuentra páginas que contienen un eslabón a otra página, y la palabra clave "url" encuentra páginas con carácteres específicos en la dirección . Aquí están unos ejemplos:

        link:tourbus.com    - encuentra páginas conectadas al sitio del TOURBUS
        url:elvis    - encuentra páginas con "elvis" en la dirección

    Con combinar cualquiera de estos términos de búsqueda, la eficiencia de su búsqueda en Alta Vista puede ir aumentando. Y aunque yo no he hablado de todas las palabras claves reservadas que Uds. pueden usar para pulir el objeto de su deseo. Si Uds. quieren aprender más acerca de la búsqueda avanzada de Alta Vista, visiten http://www.altavista.digital.com/av/content/help.htm
