NOMBRE

null, zero - sumidero de datos  

DESCRIPCIÓN

Los datos escritos en los ficheros especiales null o zero se desechan.

La lectura del fichero especial null siempre devuelve "fin de fichero (EOF)", mientras que la lectura del fichero zero siempre devuelve el caracteres \0 (NULL).

null y zero se crean normalmente con:

mknod -m 666 /dev/null c 1 3
mknod -m 666 /dev/zero c 1 5
chown root:mem /dev/null /dev/zero

 

NOTAS

Si estos dos ficheros no tienen permiso de lectura y escritura para todos los usuarios, muchos programas se comportarán de manera extraña.  

FICHEROS

/dev/null
/dev/zero  

VÉASE TAMBIÉN

mknod

(1), chown(1)
