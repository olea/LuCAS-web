PgAccess - Introducción

Qué es PgAccess?

PgAccess es una interface gráfica para el gestor de bases de datos PostgreSQL escrito por Constantin Teodorescu en el lenguaje Tcl/Tk. Permite al usuario interactuar con PostgreSQL de una manera similar a muchas aplicaciones de bases de datos para PC, con menús de opciones y diversas herramientas gráficas. Esto significa que el usuario puede evitar la linea de comandos para la mayoría de las tareas. PgAccess no cambia el modo de actuar de postgreSQL, sólo hace más fácil su uso para aquellos que esten habituados a interfaces gráficas.

Obviamente, debes tener instalado y en marcha (corriendo) PostgreSQL, y Tcl/Tk en tu sistema antes de poder usar PgAccess.

PgAccess es una aplicacion "open source" (código abierto) . El código está disponible para el usuario , y puede ser modificado. El usuario puede arreglar fallos, o cambiar la manera de operar de una función. Puede que no estés interesado en cambiar su programación , pero tienes la opción de hacerlo. Si sientes que has hecho una mejora al programa, se te anima a compartirlo con los demás.

Si no estás familiarizado en cómo se puede cambiar el software de código abierto y redistribuirlo, por favor lee ésto.

Cómo conseguir ayuda con PgAccess

La lista de correo para PgAccess es: pgsql-interfaces@postgresql.org

Si tienes alguna pregunta referente a PgAccess deberías subscribirte a la lista de la siguiente manera:

Primero subscribete a la lista enviando un email a:

pgsql-interfaces-request@postgresql.org

Envía una sola linea en el mensaje como sigue:

subscribe

En breve recibirás un mensaje empezando así:

Welcome to the pgsql-interfaces mailing list!
...

Este contendrá instrucciones de cómo puedes borrarte de la lista de correo, así que guarda el mensaje. Puede que sólo quieras hacer algunas preguntas y después dejar de recibir mensajes.

Puedes también mandar un email a Constantin Teodorescu directamente, aunque escribir a la lista a menudo produce respuestas mas rápidas.
