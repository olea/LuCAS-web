PgAccess - Comenzando

Cómo conseguir PgAccess

Si tienes esta ayuda, Probablemente ya te has bajado PgAccess. Sino, o si desea bajarte la última versión, está disponible en:

http://www.flex.ro

La página principal contiene instrucciones de qué ficheros debes bajarte para tu sistema operativo. Bájate el fichero a un directorio dónde residirá finalmente (mirar abajo).

Cómo descomprimir PgAccess

PgAccess, como la mayoría de las aplicaciones a ser bajadas, se baja en un formato comprimido para ahorrar tiempo. Debes descomprimir el archivo para poder usar la aplicación.

UNIX (Linux, BSD, IRIX, Solaris, etc.)

Los archivos serán comprimidos usando "gzip" y empaquetados usando "tar", y tienen nombres de archivo como éste:

pgaccess-n.nn.tar.gz

Notar que "n.nn" será el número de la versión actual.

Primero decide dónde quiers tener el programa. Una localización típica en sistemas UNIX es /usr/local/src/<nombre>, donde <nombre> es el nombre del programa. Para usar esta localización bate o mueve el fichero "tar.gz" al directorio /usr/local/src. Ve a ese directorio, y descomprime el archivo con el comando:

tar -zxvf pgaccess-n.nn.tar.gz

Deberías ver un lista de los archivos descomprimidos y colocados en el nuevo directorio, y entonces tendrás el directorio:

/usr/local/src/pgaccess

En ese directorio estarán todos los ficheros que estaban comprimidos en el archivo que te bajaste . Cuando tengas PgAcces funcionando, puedes borrar el fichero de extension ".tar" o ".tar.gz".

Windows

Los archivos serán comprimidos de manera que puedan ser descomprimidos con "WinZip". Sólo tienes que abrirlos con "WinZip" y el programa será extraido.

Colocando PgAccess donde pueda ser encontrado

UNIX

Para poder ejecutar PgAccess fácilmente, el archivo de programa "pgaccess.tcl" debería estar en el "PATH" del sistema. Puedes comprobar si está en el PATH tecleando:

echo $PATH
/sbin:/usr/sbin:/bin:/usr/bin:/usr/X11R6/bin:/usr/local/bin:./:/usr/local/pgsql/bin

Generalmente el directorio /usr/local/bin estará en tu PATH, y ése es un lugar muy común donde colocar programas como PgAccess. Otros directorios comunes en el PATH son /usr/bin and /usr/sbin. Simplemente cambia el fichero "pgaccess.tcl" al directorio donde prefieras.

Windows

Generalmente enlazaras PgAccess a un icono, así que sólo tienes que especificar la ruta completa al programa cuando crees el icono.

Comenzando PgAccess

La manera más fácil de llamar al programa es tecleando:

pgaccess.tcl

Si el programa ha sido colocado en un directorio que esté en el PATH, la ventana de PgAccess debería aparecer.

Comenzando desde un menú

La mayoría de los usuarios desearán enlazar el programa a un menú o un icono para poder lanzarlo usando el ratón. Aquí va un métod que funcionará en la mayoría de sistemas Linux X-Windows. Crea un fichero llamado "pgaccess" en el directorio /etc/X11/wmconfig conteniendo lo siguiente:

pgaccess name "PgAccess"
pgaccess description "postgreSQL frontend"
pgaccess mini-icon "mini-pgaccess.xpm"
pgaccess group "Applications"
pgaccess exec "pgaccess.tcl &"

Esto presupone que tienes un submenú "Applications". Podrías preferir "Programs" o cualquier otro lugar. Tambien, deberías crear un icono "mini-pgaccess.xpm" si quieres que aparezca. Puedes editar un icono existente en el directorio/usr/share/icons/mini con XPaint y renombrarlo. Cuando reinicies las X-Windows , debería haber un icono "PgAccess" en el sub-menú que hayas elegido. Pinchando en él debería lanzarse PgAccess.
