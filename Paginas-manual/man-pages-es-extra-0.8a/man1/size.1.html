NOMBRE

size - lista tamaños de las secciones y tamaño total.

 

SINOPSIS

size
[-A|-B|--format=compatibilidad] [--help] [-d|-o|-x|--radix=número] [--target=nombrebfd] [-V|--version] fichobj...
 

DESCRIPCIÓN

La utilidad size de GNU lista los tamaños de las secciones ---y el tamaño total--- para cada uno de los ficheros objeto fichobj en su lista de argumentos. De forma predeterminada, se genera una línea de salida para cada fichero objeto o cada módulo de un archivo.

 

OPCIONES

-A
-B
--format=compatibilidad
Empleando una de estas opciones, se puede escoger si la salida del size de GNU se parecerá a la salida del size de System V (empleando `-A', o `--format=sysv'), o a la del size de Berkeley (empleando `-B', o `--format=berkeley'). El formato predeterminado es el de una línea similar al de Berkeley.
--help
Muestra un resumen de los argumentos y opciones que se aceptan.
-d
-o
-x
--radix=número
Mediante una de estas opciones, se puede controlar si el tamaño de cada sección se da en base diez (`-d', o `--radix=10'); octal (`-o', o `--radix=8'); o hexadecimal (`-x', o `--radix=16'). En `--radix=número', sólo se admiten los tres valores 8, 10 ó 16. El tamaño total se da siempre en dos bases: decimal y hexadecimal para la salida `-d' o `-x', u octal y hexadecimal si se usa `-o'.
--target=nombrebfd
Se puede especificar nu formato de código objeto particular para fichobj en nombrebfd. Esto puede no ser necesario; size puede automáticamente reconocer muchos formatos. Consulte objdump(1) para más información sobre cómo ver los formatos disponibles.
-V
--version
Muestra información del número de versión del propio size.
